🚀 Website Setup Guide

Get your website protected by the WAF in minutes

⏱️~5 min setup
🔧No code changes
🛡️Instant protection
1

Register Your Website

Go to My Sites and click "+ Onboard Website". Enter your domain, backend URL, and choose a WAF mode.

💡 Tip: Start with DETECTION mode to monitor traffic before blocking. This lets you identify false positives without disrupting real users.
2

Point DNS to the WAF

Update your DNS records so your domain points to the WAF server's IP address instead of your backend directly.

# Change DNS A Record:
# BEFORE: shop.example.com → your-server-ip
# AFTER:  shop.example.com → WAF-server-ip

# Or use ngrok for instant tunneling:
ngrok http 8080 --hostname=your-domain.ngrok.app
⚠️ Important: If using a CDN like Cloudflare, set the WAF server IP as the origin. The WAF should sit between the CDN and your backend.
3

Lock Down Your Backend

Configure your server's firewall to only accept connections from the WAF server IP. This prevents attackers from bypassing the WAF.

# UFW (Ubuntu/Debian)
sudo ufw allow from WAF_IP to any port 80
sudo ufw allow from WAF_IP to any port 443
sudo ufw deny 80
sudo ufw deny 443

# iptables
iptables -A INPUT -p tcp -s WAF_IP --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 80 -j DROP
4

Test It

Verify everything works by sending normal and malicious requests.

# Normal request (should work normally)
curl https://your-domain.com/

# Test SQL injection blocking (should get 403 in BLOCKING mode)
curl "https://your-domain.com/?q=' OR 1=1--"

# Test XSS blocking
curl "https://your-domain.com/?q=<script>alert(1)</script>"

# Test path traversal blocking
curl "https://your-domain.com/../../etc/passwd"
5

Monitor & Tune

Watch the Live Monitor for traffic. Review Security Insights for recommendations. Check Bot Activity to see who's visiting your site.

📊 Recommended: Monitor for at least 24 hours before switching to BLOCKING mode. Use the Alerts tab to get notified of anomalies automatically.
6

Enable Blocking & Add Rules

Switch to BLOCKING mode from My Sites. Then use Header Blacklist to block specific User-Agents (scanners, scrapers), and the Rule Sandbox below to test custom patterns.
